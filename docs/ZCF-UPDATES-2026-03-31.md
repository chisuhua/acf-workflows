# ZCF 命令集更新报告

**更新时间**: 2026-03-31  
**更新人**: DevMate  
**更新范围**: `~/.agents/commands/zcf/` 目录

---

## 更新背景

基于 P0 实施后的质检员审查，发现 zcf 命令集存在以下问题：
1. **角色定义缺失** — 编码助手不知道自己的身份和反馈对象
2. **与 acf-workflow 术语不一致** — 可能导致混淆
3. **task-review 输出未要求给 DevMate** — 评审报告格式不统一
4. **缺少架构问题反馈流程** — 编码助手发现问题时不知道如何处理
5. **分支策略未定义** — 编码助手不知道是否应该创建分支

**解决方案**: 实施替代方案（而非原 P1 的 task-handoff.md + start-multi-agent.sh）

---

## 更新内容

### 1. `task-review.md` 更新

**新增章节**:
- **编码助手角色定义** — 明确身份、职责、反馈对象、禁止行为
- **Git 分支策略** — 引用 acf-workflow §2.5，明确分支命名和评审流程
- **合规检查集成** — 要求评审前运行 `check-compliance.sh`

**更新章节**:
- **输出报告格式** — 标准化为 4 个章节（给 DevMate 看）
  1. 任务执行结果（产出物、Git 提交、测试结果、合规检查）
  2. 架构一致性检查（一致项、偏差）
  3. 给 DevMate 的行动建议（需要确认的事项、需要执行的命令）
  4. 下一步决策（下一任务、建议行动）

**偏差级别定义**（标准化）:
| 级别 | 定义 | 处理策略 |
|------|------|----------|
| **轻微** | 向后兼容变更，不影响其他模块 | 立即更新文档或累积更新 |
| **中等** | 需要记录的技术选型变更 | 更新架构文档，可能需要 ADR |
| **严重** | 架构违规或破坏性变更 | 暂停，DevMate 审查，必须 ADR |

---

### 2. `templates/task-review-template.md` 更新

**更新内容**:
- 添加角色定义（编码助手 → DevMate）
- 标准化模板结构（4 个必选章节）
- 添加填写指南（编码助手必读）
- 添加偏差级别定义表
- 添加合规检查要求
- 添加 Git 分支策略说明

**模板结构**:
```markdown
# {{TASK_ID}} 完成评审报告（给 DevMate）

## 一、任务执行结果
- 产出物清单
- Git 分支和提交
- 测试结果 + 覆盖率
- 合规检查（必须通过）

## 二、架构一致性检查
- 一致项
- 偏差（如无写"无"）

## 三、给 DevMate 的行动建议
- 需要 DevMate 确认的事项
- 需要 DevMate 执行的命令
- 文档更新建议
- 更新策略

## 四、下一步决策
- 下一任务
- 前置条件检查
- 建议行动（选项 A/B/C）
```
