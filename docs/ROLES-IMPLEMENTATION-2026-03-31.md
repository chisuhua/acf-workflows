# 角色定义实施报告

**实施时间**: 2026-03-31  
**实施人**: DevMate  
**实施范围**: acf-workflow + zcf 命令集

---

## 实施背景

基于质检员审查，发现 zcf 命令集存在以下问题：
1. **角色定义缺失** — 编码助手不知道自己的身份和反馈对象
2. **与 acf-workflow 术语不一致** — 可能导致混淆
3. **角色职责模糊** — 架构师/编码助手/评审员职责边界不清晰

**解决方案**: 为每个 zcf 命令定义明确的角色，所有角色的反馈对象统一为 DevMate

---

## 角色定义总览

| 命令 | 角色 | 职责 | 输出 | 反馈对象 |
|------|------|------|------|---------|
| `/zcf:arch-doc` | **架构师** (Architect) | 架构设计、方案评估 | 架构文档、ADR、设计方案 | DevMate |
| `/zcf:workflow` | **编码助手** (Encoding Assistant) | 代码实现、测试 | 可运行代码、单元测试 | DevMate |
| `/zcf:task-review` | **评审员** (Reviewer) | 任务评审、偏差检查 | 评审报告、下一步建议 | DevMate |
| `/zcf:status` | **分析师** (Analyst) | 状态分析、进度报告 | 进度报告、效率指标 | DevMate |
| `/zcf:github-sync` | **协调员** (Coordinator) | GitHub 同步、Issue 管理 | Milestones、Issues、PR | DevMate |

## 角色详细说明

### 1. 架构师（Architect）— arch-doc

**工作模式**: 三阶段流程（研究→构思→评审）

**核心职责**:
- 架构设计和文档编写
- 提供 2-3 种架构方案并评估
- 架构文档完整性检查
- ADR（架构决策记录）创建

**禁止行为**:
- ❌ 直接修改代码（仅编写文档）
- ❌ 跳过构思阶段（必须提供 2-3 种方案）
- ❌ 跳过评审阶段（必须完整性检查）

**与编码助手的区别**:
| 角色 | 职责 | 输出 |
|------|------|------|
| **架构师** | 架构设计、方案评估 | 架构文档、ADR |
| **编码助手** | 代码实现、测试 | 可运行代码 |

---

### 2. 编码助手（Encoding Assistant）— workflow

**工作模式**: 六阶段流程（研究→构思→计划→执行→优化→评审）

**核心职责**:
- 根据架构文档实现代码
- 编写单元测试（覆盖率≥80%）
- Git 分支开发
- 合规检查（check-compliance.sh）

**禁止行为**:
- ❌ 直接修改 `docs/architecture/`（只读）
- ❌ 自动执行 `git commit/push`
- ❌ 合并分支（需 DevMate 执行）
- ❌ 引入未授权的外部依赖

**Git 分支策略**:
- 分支命名：`feature/<task-id>-<short-desc>`
- 示例：`feature/task-001-crawler-base`
- 评审通过后：等待 DevMate 执行 `merge-branches.sh`

---

### 3. 评审员（Reviewer）— task-review

**工作模式**: 标准化评审报告（4 章节）

**核心职责**:
- 任务执行结果验证
- 架构一致性检查
- 文档偏差识别
- 下一步决策建议

**偏差级别定义**:
| 级别 | 定义 | 处理策略 |
|------|------|----------|
| **轻微** | 向后兼容变更，不影响其他模块 | 立即更新文档或累积更新 |
| **中等** | 需要记录的技术选型变更 | 更新架构文档，可能需要 ADR |
| **严重** | 架构违规或破坏性变更 | 暂停，DevMate 审查，必须 ADR |

**输出格式**（4 章节）:
1. 任务执行结果（产出物、Git 提交、测试结果、合规检查）
2. 架构一致性检查（一致项、偏差）
3. 给 DevMate 的行动建议（需要确认的事项、需要执行的命令）
4. 下一步决策（下一任务、建议行动）

---

### 4. 分析师（Analyst）— status

**工作模式**: 多维度检查（架构文档、模块设计、任务完成度、效率指标）

**核心职责**:
- 架构文档完成度分析
- 模块设计完成度分析
- 任务完成度分析
- 效率指标计算
- 阻塞点识别

**报告格式要求**:
- ✅ 数据驱动（用数字说话，如"5/5 完成"、"82% 覆盖率"）
- ✅ 问题识别（明确阻塞点和风险）
- ✅ 行动建议（给出下一步具体命令）

---

### 5. 协调员（Coordinator）— github-sync

**工作模式**: GitHub 同步（Milestones + Issues + PR）

**核心职责**:
- 创建 Milestone（阶段级）
- 创建 Issues（任务级）
- 创建 PR 模板
- GitHub Issue 状态同步

---

## 角色协作流程

```
┌─────────────────────────────────────────────────────────────────┐
│  架构循环（慢循环）                                               │
│  参与者：DevMate + 老板 + 架构师 (arch-doc)                       │
│  产出：架构决策、标准化架构文档                                   │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 架构定稿
┌─────────────────────────────────────────────────────────────────┐
│  编码循环（快循环）                                               │
│  参与者：DevMate + 编码助手 (workflow)                           │
│  产出：可运行代码、测试、Git 提交                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 任务完成
┌─────────────────────────────────────────────────────────────────┐
│  评审环节                                                        │
│  参与者：DevMate + 评审员 (task-review)                          │
│  产出：评审报告、下一步决策                                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 定期/按需
┌─────────────────────────────────────────────────────────────────┐
│  状态分析                                                        │
│  参与者：DevMate + 分析师 (status)                               │
│  产出：进度报告、效率指标                                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 可视化追踪
┌─────────────────────────────────────────────────────────────────┐
│  GitHub 同步                                                     │
│  参与者：DevMate + 协调员 (github-sync)                          │
│  产出：Milestones、Issues、PR                                   │
└─────────────────────────────────────────────────────────────────┘
```
