# ACF 工作流阶段审查流程

**版本**: v1.0  
**创建时间**: 2026-03-30  
**适用范围**: 所有使用 ACF 工作流的项目

---

## 🎯 审查目标

每个阶段结束时进行架构审查，决定：
1. **继续快循环** — 进入下一阶段
2. **切换到慢循环** — 架构调整

---

## 📋 审查时机

| 时机 | 触发条件 | 参与人 |
|------|---------|--------|
| **阶段完成** | 阶段任务 100% 完成 | DevMate + 老板 |
| **架构问题** | 发现严重架构缺陷 | DevMate + 老板 |
| **需求变更** | 业务范围变化 >30% | DevMate + 老板 |

---

## 🔍 审查检查清单

### 1. 架构一致性（5 项）

| 检查项 | 检查内容 | 通过标准 |
|--------|---------|---------|
| **模块结构** | 代码组织是否符合架构文档 | 模块划分一致 |
| **接口设计** | 接口是否遵循架构定义 | 接口签名一致 |
| **数据流** | 数据流是否符合架构设计 | 流向正确 |
| **依赖关系** | 依赖方向是否符合架构 | 无循环依赖 |
| **错误处理** | 错误处理是否遵循策略 | 符合错误处理策略 |

### 2. 实现质量（4 项）

| 检查项 | 检查内容 | 通过标准 |
|--------|---------|---------|
| **测试覆盖率** | 单元测试覆盖率 | >80% |
| **测试通过率** | 所有测试通过 | 100% |
| **代码质量** | 代码规范、可读性 | 良好 |
| **文档完整** | 文档字符串、注释 | 完整 |

### 3. 架构调整需求（4 项）

| 评估项 | 评估内容 | 切换到慢循环条件 |
|--------|---------|-----------------|
| **需求变更** | 是否有新需求 | 是 |
| **技术选型** | 是否需要更换技术 | 是 |
| **架构缺陷** | 是否发现架构缺陷 | 严重 |
| **扩展需求** | 是否需要新增模块 | 是 |

---

## 🔀 决策流程

```
阶段完成
    ↓
执行阶段审查（DevMate + 老板）
    ↓
填写审查报告
    ↓
发现架构问题？
    ├─ 无/轻微 → ✅ 继续快循环（下一阶段）
    ├─ 中等 → ⚠️ 记录技术债务，继续快循环
    └─ 严重 → ❌ 切换到慢循环（架构调整）
```

---

## 📊 问题分级标准

| 级别 | 描述 | 处理方式 |
|------|------|---------|
| **轻微** | 不影响架构，代码层面可修复 | 记录，快循环中修复 |
| **中等** | 影响局部设计，不影响整体架构 | 记录技术债务，优先修复 |
| **严重** | 影响系统架构，需要重新设计 | 切换到慢循环 |

## 🔄 切换到慢循环的条件

满足以下**任一条件**即切换到慢循环：

1. **架构缺陷** — 发现严重架构设计问题
2. **技术选型变更** — 需要更换核心技术栈
3. **需求重大变更** — 业务范围变化 >30%
4. **新增模块** — 需要新增架构层或核心模块
5. **老板决策** — 老板要求重新审查架构

## 🎯 新项目启动流程

### Step 1: 创建项目目录

```bash
mkdir -p /workspace/<project-name>/{.acf/{status,temp,config},src,tests,docs/architecture}
mkdir -p /workspace/mynotes/<project-name>/docs/architecture/{decisions,reviews,plans}
```

### Step 2: 创建项目配置文件

```bash
# 复制领域配置模板
cp /workspace/acf-workflow/templates/domains-config.yaml \
   /workspace/mynotes/<project-name>/docs/architecture/plans/

# 复制触发器配置
cp /workspace/acf-workflow/config/acf-triggers.yaml \
   /workspace/<project-name>/.acf/config/
```

### Step 3: 启动慢循环（架构设计）

```
1. DevMate + 老板讨论架构需求
2. DevMate 创建系统架构设计
3. 老板评审确认
4. 同步到编码仓库
```

### Step 4: 开始快循环（阶段 1）

```
1. 创建阶段 1 任务计划
2. 执行 Task 001, Task 002, ...
3. 阶段完成后执行阶段审查
4. 决定：继续快循环 vs 切换到慢循环
```
