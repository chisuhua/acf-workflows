## 修复内容

### 修复 1: 统一角色术语

**之前**:
```
| 角色 | 身份 |
|------|------|
| DevMate | 技术合伙人 |
| OpenCode | 编码架构师 |
| Claude Code | 编码架构师 |
```

**修复后**:
```
| 角色 | 身份 | 触发方式 |
|------|------|---------|
| DevMate | 技术合伙人/终端操作员 | 直接调用 acf-* Skills |
| 架构师 | OpenCode/Claude Code (运行 arch-doc) | DevMate 通过 ACP session 输入 |
| 编码助手 | OpenCode/Claude Code (自主执行任务) | OpenCode 自主决定 |
| 评审员 | OpenCode/Claude Code (运行 task-review) | DevMate 通过 ACP session 输入 |
| 分析师 | OpenCode/Claude Code (运行 status) | DevMate 通过 ACP session 输入 |
```

**修改文件**:
- 1.3 节：角色与命令对应关系（统一术语）
- 2.1 节：角色矩阵（统一术语）
- 2.2 节：/zcf/ 技能列表（按角色分类）

---

### 修复 2: 明确 acf-executor 执行流程

**新增 2.4 节**:

```
1. DevMate 调用 acf-executor
   skill_use acf-executor task="Task 001: ..."

2. acf-executor 启动 OpenCode（编码助手角色）
   - 加载 templates/task-prompt.md
   - sessions_spawn(runtime="acp", agentId="opencode")
   - 更新状态机：IDLE → EXECUTING

3. OpenCode 自主执行代码（内部可能调用 /zcf/workflow，可选）
   - 读取架构文档
   - 实现代码
   - 编写测试
   - 运行合规检查

4. OpenCode 完成任务
   - 更新状态机：EXECUTING → WAITING_REVIEW
   - 等待 DevMate 评审

5. DevMate 运行评审（通过 ACP session 输入）
   /zcf/task-review "Task 001 完成"
   → 唤醒评审员角色 → 输出报告
```

**关键点**:
- OpenCode 执行方式是**自主决定**的
- DevMate 作为**终端操作员**，通过 ACP session 输入 `/zcf/task-review`

---

### 修复 4: 明确架构循环→编码循环切换

**新增 2.6 节**:

```
1. 架构师 (arch-doc) 完成架构文档
   输出：docs/architecture/YYYY-MM-DD-xxx.md

2. DevMate 对比评审（自己执行，不调用技能）
   - 读取 mynotes 提案草稿
   - 读取编码仓库标准化文档
   - 对比一致性

3. 一致 → acf-sync 同步
   skill_use acf-sync

4. acf-executor 进入编码循环
   skill_use acf-executor task="Task 001: ..."
```

**DevMate 对比检查清单**:
- [ ] 架构文档结构一致
- [ ] 模块边界一致
- [ ] 接口定义一致
- [ ] 技术选型一致
- [ ] ADR 约束一致

## 完整交互流程（修复后）

```
┌─────────────────────────────────────────────────────────────────┐
│  架构循环                                                        │
│  1. 老板 + DevMate 讨论架构 → mynotes/                          │
│  2. DevMate → acf-executor (架构师角色)                          │
│  3. OpenCode 运行 /zcf/arch-doc → 编码仓库/docs/architecture/   │
│  4. DevMate 对比 (mynotes vs 编码仓库) → 一致？                  │
│  5. 是 → acf-sync 同步                                           │
│  6. 否 → 返回步骤 1 调整（通知架构师更新）                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  编码循环                                                        │
│  1. DevMate → acf-executor (编码助手角色)                        │
│  2. OpenCode 自主执行代码（可能调用 /zcf/workflow）              │
│  3. OpenCode 完成 → WAITING_REVIEW                               │
│  4. DevMate → ACP session 输入 /zcf/task-review                  │
│  5. OpenCode 运行评审员 → 输出报告                               │
│  6. DevMate 查看报告 → acf-flow → 下一任务                       │
└─────────────────────────────────────────────────────────────────┘
```
